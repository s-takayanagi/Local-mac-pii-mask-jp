# Security Policy

## サポート対象バージョン

本プロジェクトは技術プレビュー版です。現時点でセキュリティ修正の対象となるのは `main` ブランチの最新コミットのみです。タグ付きリリースごとのサポートは、正式版リリース時に改めて方針を定めます。

| バージョン | サポート状況 |
|---|---|
| `main`（最新） | ✅ 修正対象 |
| それ以前 | ❌ 対象外 |

---

## 脆弱性の報告方法

セキュリティ上の脆弱性を発見された場合は、**公開の GitHub Issue には投稿せず**、以下のいずれかの方法で非公開にご連絡ください。

### 推奨: GitHub Private Vulnerability Reporting

1. リポジトリの **Security** タブを開く
2. **Report a vulnerability** をクリック
3. 再現手順・影響範囲・想定リスクを記入して送信

### 代替手段

GitHub の機能が使えない場合は、リポジトリオーナー（`@s-takayanagi`）宛に GitHub の DM もしくはメンションで「非公開で連絡したい旨」をお知らせください。別途連絡手段をご案内します。

---

## 報告に含めていただきたい情報

- 脆弱性の概要（想定される攻撃シナリオ・影響範囲）
- 再現手順（できるだけ最小の再現コード・入力例）
- 対象バージョン / コミットハッシュ / 実行環境
- 発見時点で判明している緩和策（あれば）
- 公開可能な開示期限のご希望（なければ相談ベースで調整）

## 脅威モデル上の前提

本ツールは以下を前提に設計されています。報告前にご参照ください。

- **PII を含む入力ファイルはローカルで処理されます。** 外部送信は LM Studio（`localhost:1234`）への推論リクエストのみで、それ以外の外部通信は発生しません。
- **マスキング精度は 100% を保証しません。** 見落とし（false negative）は脆弱性ではなく、既知の制約事項として扱います。ただし特定パターンで **系統的に** 検出されない・ログ等に平文 PII が残るといった実装欠陥は脆弱性として扱います。
- **LM Studio 側のセキュリティは本プロジェクトのスコープ外です。** LM Studio 自体の脆弱性は提供元（Element Labs, Inc.）にご報告ください。
- **Docker / macOS / Claude Desktop の脆弱性も本プロジェクトのスコープ外です。** 本プロジェクトのコード・設定に起因する問題のみを対象とします。

---

## 既知の制約事項

以下は既知の制約であり、脆弱性ではなく設計上のトレードオフとして扱います。

- MCP サーバー機能は未検証です。Claude Desktop 連携では認証・レート制限を実装していません（ローカル stdio 前提）。
- Docker コンテナは `host.docker.internal` 経由でホストの LM Studio に接続します。コンテナ外への意図しない通信を防ぐファイアウォール設定は利用者側で行ってください。
