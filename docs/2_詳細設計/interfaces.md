# インターフェース設計

## 1. Streamlit Web UI（`ui/streamlit_app.py`）

### アクセス方法

```
http://localhost:8501
```

### 画面構成

```
┌─────────────────────────────────────────────────────┐
│  サイドバー                                          │
│  ├── LM Studio 接続状態（接続中 / 未接続）            │
│  ├── モデル選択（LM Studio から取得したモデル一覧）    │
│  │   └── LFM2 モデル選択時: 特化型モードの案内を表示  │
│  ├── 処理レイヤー（Layer 1〜4 の ON/OFF チェックボックス）│
│  └── 除外する項目（PII カテゴリ別チェックボックス）    │
│       氏名 / 住所 / 電話番号 / メール / 会社名 等      │
├─────────────────────────────────────────────────────┤
│  メインエリア（処理前）                               │
│  ├── フォルダパス入力                                 │
│  └── マスキング開始ボタン（未接続・未選択時は disabled）│
│                                                     │
│  メインエリア（処理中）                               │
│  ├── プログレスバー（ファイル単位）                    │
│  ├── リアルタイム置換ログ（直近50件テーブル）           │
│  └── システムログ（高さ制限付きスクロール表示）         │
│                                                     │
│  メインエリア（完了後）                               │
│  ├── サマリー（ファイル数・総置換件数・エラー有無）     │
│  ├── ファイル別詳細（expander）                       │
│  │   ├── レイヤー別検出件数（4列メトリクス）            │
│  │   ├── 未マスクの可能性がある項目の警告（docx のみ）  │
│  │   ├── 処理中エラー一覧                             │
│  │   └── 置換詳細テーブル                             │
│  ├── 置換ログ（コピー用テキスト）                     │
│  └── システムログ（デバッグ用 expander）              │
└─────────────────────────────────────────────────────┘
```

### 動作

1. サイドバーで LM Studio 接続を確認
2. モデルを選択（LFM2 は自動で専用モードに切り替わる）
3. 処理レイヤーと除外PII項目を設定
4. フォルダパスを入力（コンテナ内パス: `/data/...`）
5. 「マスキング開始」をクリック
6. 完了後、ファイル別レイヤー検出件数・置換詳細・システムログを確認

### ログ管理（`ui/log_handler.py`）

`SessionStateLogHandler` が root ロガーに attach され、処理中のログを
`st.session_state["system_logs"]` に蓄積してリアルタイム表示する。
処理完了後に uninstall され、次回実行時に再 install される（重複防止ガード付き）。

## 2. MCP サーバー（`mcp/server.py`）

Claude Desktop から直接 PII マスキングを実行するための MCP（Model Context Protocol）サーバー。

### 起動方法

```bash
python main.py --mode mcp
```

Claude Desktop の設定ファイルに登録することで、stdio 経由で自動起動される。

### 提供ツール

#### `mask_text_tool(text: str) -> str`

テキスト文字列をマスキングする。

| 引数 | 型 | 説明 |
|------|----|------|
| `text` | `str` | マスキング対象のテキスト |

**戻り値**: マスク済みテキスト文字列

---

#### `mask_file(file_path: str) -> str`

単一ファイルをマスキングする。

| 引数 | 型 | 説明 |
|------|----|------|
| `file_path` | `str` | `.xlsx` / `.pptx` / `.docx` ファイルのパス |

**戻り値**: 成功メッセージ（出力パス・置換件数を含む）またはエラーメッセージ

**出力ファイル**: `{元ファイル名}_masked.{拡張子}`

---

#### `mask_folder(folder_path: str) -> str`

フォルダ内の対応ファイルをすべてマスキングする。

| 引数 | 型 | 説明 |
|------|----|------|
| `folder_path` | `str` | 処理対象フォルダのパス |

**戻り値**: 全ファイルの処理結果サマリー

---

## 3. CLI モード

バッチ処理向けのコマンドラインインターフェース。

```bash
python main.py --mode cli --folder /data/path/to/folder
```

| オプション | 説明 |
|-----------|------|
| `--mode cli` | CLI モードで起動 |
| `--folder <path>` | 処理対象フォルダのパス |

## 4. LM Studio API（内部）

Layer 3 / Layer 4 で使用する内部 API。

| 項目 | 値 |
|------|----|
| エンドポイント | `http://host.docker.internal:1234/v1/chat/completions` |
| メソッド | POST |
| 形式 | OpenAI 互換 JSON |
| レスポンス | `choices[0].message.content`（JSON 文字列） |
