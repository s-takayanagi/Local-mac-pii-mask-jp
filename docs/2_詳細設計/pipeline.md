# マスキングパイプライン設計

## 概要

テキストは 4 つのレイヤーを順番に通過する。各レイヤーは前レイヤーの出力を受け取り、さらに PII を検出・置換する。

```
入力テキスト
    │
    ▼
Layer 1: 正規表現マッチング
    │
    ▼
Layer 2: NER（固有表現認識）
    │
    ▼
Layer 3: LLM マスキング
    │
    ▼
Layer 4: LLM レビュー
    │
    ▼
マスク済みテキスト
```

---

## Layer 1 - 正規表現マッチング（`core/layer1_regex.py`）

パターンが明確な PII を高速に検出・置換する。

| 対象 | タグ | パターン例 |
|------|------|-----------|
| 携帯電話 | `[電話番号]` | `0[789]0[-−]\d{4}[-−]\d{4}` |
| 固定電話 | `[電話番号]` | `0\d{1,4}[-−]\d{2,4}[-−]\d{3,4}` |
| メール（.co.jp 形式含む） | `[メール]` | `[\w.+-]+@[\w-]+\.(?:[a-zA-Z]{2,}\.)*[a-zA-Z]{2,}` |
| 郵便番号 | `[郵便番号]` | `\d{3}[-−]\d{4}` |
| マイナンバー等 | `[識別番号]` | 前後が非数字の 12 桁数字列 |
| URL | `[URL]` | `https?://[^\s\u3000-\u9fff\uff00-\uffef]+`（全角文字で打ち切り） |
| 住所（都道府県+番地） | `[住所]` | `(?:北海道\|東京都\|...)` + 市区町村 + 番地 ± 建物名 |

**特徴**: コンパイル済み正規表現による高速処理。クラウド・モデル不要。
住所パターンは空白・句読点で境界を制限し（`[^\s。、,，]{1,N}`）過検出を抑制。

---

## Layer 2 - NER（`core/layer2_ner.py`）

日本語 NER モデル `ja_ginza_electra`（spaCy）で固有表現を検出する。

| GiNZA エンティティ | タグ | 備考 |
|------------------|------|------|
| `Person` / `PERSON` | `[氏名]` | 人名 |
| `City` / `Province` / `Address` | `[住所]` | 地名・住所 |
| `Organization` / `Company` | `[会社名]` | 組織・企業名 |
| `Phone` | `[電話番号]` | Layer 1 漏れの補完 |
| `Email` | `[メール]` | `@` を含まない場合は誤検知として除外 |

**処理の注意点**
- 文字位置の後方イテレーションで置換（位置ずれ防止）
- 都道府県名のみ（行政区画）は `_is_admin_unit()` で除外してノイズを抑制
- `@` を含まないエンティティを `[メール]` と誤分類しないチェックあり

---

## Layer 3 - LLM マスキング（`core/layer3_llm.py`）

LM Studio 経由で文脈依存の PII を検出する。選択モデルに応じて2つの動作モードがある。

### 汎用 LLM モード（デフォルト）

**システムプロンプト（MASKER_SYSTEM）の検出対象**
- 氏名・住所・会社名・識別番号のほか、電話・メール・郵便番号・URL（Layer1/2 の漏れ補完）
- すでにタグ付けされた箇所は変更しない
- 出力: `{ "masked_text": "...", "replacements": [...] }`

| パラメータ | 値 |
|-----------|---|
| temperature | `0.05` |
| max_tokens | `4096` |
| タイムアウト | `120` 秒 |

### LFM2 専用モード（モデル名に "lfm2" を含む場合）

| パラメータ | 値 |
|-----------|---|
| システムプロンプト | `Extract <address>, <company_name>, <email_address>, <human_name>, <phone_number>` |
| temperature | `0`（greedy decoding） |
| 出力形式 | `{"human_name": [...], "address": [...], ...}` を `replacements` に変換 |
| 部分文字列競合対策 | エンティティを長さ降順で処理 |
| 重複出現対応 | 同一エンティティを全出現箇所で置換 |

---

## Layer 4 - LLM レビュー（`core/layer3_llm.py`）

Layer 3 の出力に対して第 2 の LLM パスを実行し、見落としの補足と過検出の修正を行う。

**入力**: 常に【原文】と【マスク済みテキスト】の2セクション形式で送信
（原文参照により過検出の判断精度を向上）

**確認ポイント**
1. マスク済みテキストに残存している人名・住所の断片・会社名
2. 原文と比較した過検出（一般名詞・部署名・法律上の甲乙等）を元に戻す

**出力形式**

```json
{
  "final_text": "最終マスク済みテキスト",
  "additional": [
    { "original": "田中", "tag": "[氏名]" }
  ],
  "confidence": 0.95
}
```

`confidence` は 0.0〜1.0 のスコアで、マスキングの完全性を示す。
LFM2 モードでは固定値 `0.95` を返す。

---

## パイプライン統括（`core/pipeline.py`）

各レイヤーをシーケンシャルに呼び出し、最終的な `MaskResult` を返す。

```python
def mask_text(
    text: str,
    model: str,
    lm_studio_url: str,
    enabled_layers: set[str] | None = None,  # None = 全レイヤー有効
    excluded_tags: set[str] | None = None,   # 除外タグ（全レイヤーに伝播）
) -> MaskResult: ...
```

**フォールバック挙動**

| 失敗レイヤー | フォールバック先 | confidence |
|------------|--------------|-----------|
| Layer 3 | Layer 1+2 の結果をそのまま返却 | 0.7 |
| Layer 4 | Layer 3 の結果をそのまま返却 | 0.8 |

**除外タグ（`excluded_tags`）の動作**
- Layer 1/2: 対象タグのパターンをスキップ
- Layer 3/4: LLM が masked したタグを `_revert_excluded()` で元の文字列に戻す
  - 複数出現がある場合は `reference_text`（マスク前テキスト）の出現順に復元
