# マスキングパイプライン設計

## 概要

テキストは 4 つのレイヤーを順番に通過する。各レイヤーは前レイヤーの出力を受け取り、さらに PII を検出・置換する。

```
入力テキスト
    │
    ▼
Layer 1: 正規表現マッチング
    │
    ▼
Layer 2: NER（固有表現認識）
    │
    ▼
Layer 3: LLM マスキング
    │
    ▼
Layer 4: LLM レビュー
    │
    ▼
マスク済みテキスト
```

---

## Layer 1 - 正規表現マッチング（`core/layer1_regex.py`）

パターンが明確な PII を高速に検出・置換する。

| 対象 | タグ | パターン例 |
|------|------|-----------|
| 携帯電話 | `[電話番号]` | `0[789]0[-−]\d{4}[-−]\d{4}` |
| 固定電話 | `[電話番号]` | `0\d{1,4}[-−]\d{1,4}[-−]\d{4}` |
| メール | `[メール]` | `[\w.+-]+@[\w-]+\.(?:[a-zA-Z]{2,}\.)*[a-zA-Z]{2,}` |
| 郵便番号 | `[郵便番号]` | `\d{3}[-−]\d{4}` |
| マイナンバー | `[識別番号]` | 12 桁の数字列 |
| URL | `[URL]` | `https?://[^\s\u3000-\u9fff\uff00-\uffef]+` |

**特徴**: コンパイル済み正規表現による高速処理。クラウド・モデル不要。

---

## Layer 2 - NER（`core/layer2_ner.py`）

日本語 NER モデル `ja_ginza_electra`（spaCy）で固有表現を検出する。

| GiNZA エンティティ | タグ | 備考 |
|------------------|------|------|
| `Person` | `[氏名]` | 人名 |
| `GPE` / `LOC` | `[住所]` | 地名・住所（都道府県は除外） |
| `ORG` | `[会社名]` | 組織・企業名 |

**処理の注意点**
- 文字位置の後方イテレーションで置換（位置ずれ防止）
- 都道府県名（行政区画）は除外してノイズを抑制

---

## Layer 3 - LLM マスキング（`core/layer3_llm.py`）

Qwen2.5-7B（LM Studio 経由）で文脈依存の PII を検出する。

**LLM への指示（システムプロンプト骨子）**
- 氏名（フル・略称含む）、住所断片、会社名、ID 番号を検出
- すでにタグ付けされた箇所は変更しない
- 過検出を避ける
- 出力は JSON 形式: `{ "masked_text": "...", "replacements": [...] }`

**API リクエスト設定**

| パラメータ | 値 |
|-----------|---|
| モデル | `qwen2.5-7b-instruct`（可変） |
| temperature | `0.05`（低ランダム性） |
| max_tokens | `4096` |
| タイムアウト | `120` 秒 |

---

## Layer 4 - LLM レビュー（`core/layer3_llm.py`）

Layer 3 の出力に対して第 2 の LLM パスを実行し、見落とした PII を補足する。

**検出対象（残留 PII）**
- 氏名の断片
- 住所の一部
- ブランド名に紛れた個人情報

**出力形式**

```json
{
  "final_text": "マスク済みテキスト",
  "additional": [
    { "original": "田中", "tag": "[氏名]" }
  ],
  "confidence": 0.95
}
```

`confidence` は 0.0〜1.0 のスコアで、マスキングの完全性を示す。

---

## パイプライン統括（`core/pipeline.py`）

各レイヤーをシーケンシャルに呼び出し、最終的な `MaskResult` を返す。

```python
def mask_text(text: str) -> MaskResult:
    text = layer1_regex.mask(text)
    text = layer2_ner.mask(text)
    result = layer3_llm.mask_and_review(text)
    return result
```

LM Studio が未起動の場合は Layer 1・2 のみで処理を継続する（フォールバック）。
